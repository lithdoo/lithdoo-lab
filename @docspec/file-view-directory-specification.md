# File-View 目录结构规范

## 概述

`file-view` 目录是一套用于文件展示和管理的标准化目录结构规范。该目录以 `[FILE_VIEW]` 为文件名前缀，用于标识符合该规范的专用存储区域。

本规范定义了一套完整的文件组织规则，包括特殊文件类型、隐藏机制以及元数据描述标准，旨在为文件浏览、预览和索引提供统一的数据结构支持。

## 目录命名规则

所有符合 `file-view` 规范的目录必须以 `[FILE_VIEW]` 作为文件名前缀。这一命名约定用于区分普通存储目录与遵循本规范的专用目录，便于程序在扫描文件系统时快速识别和筛选目标目录。

示例目录命名：
- `[FILE_VIEW] 视频库`
- `[FILE_VIEW] 图片集`
- `[FILE_VIEW] 文档库`

## 特殊文件类型

除常规文件和文件夹外，`file-view` 目录支持以下三种特殊文件类型：

| 文件类型 | 命名模式 | 用途 |
|---------|---------|------|
| 隐藏目录/文件 | `[hide]` 前缀 | 标记不向用户展示的私有内容 |
| 缩略图文件 | `{filename}.thumb.{ext}` | 目标文件的可视化预览 |
| 元数据文件 | `{filename}.meta.toml` | 目标文件的信息描述 |

### 隐藏目录与隐藏文件

以 `[hide]` 开头的目录或文件被视为隐藏对象。这些对象在标准的文件浏览界面中不会向终端用户展示，但仍然存在于文件系统中，可通过特殊操作或管理接口访问。

隐藏机制适用于以下场景：
- 存储临时文件或缓存数据
- 保存内部处理所需的辅助文件
- 存放用户不希望在常规浏览时看到的内容

### 缩略图文件

缩略图文件采用 `{filename}.thumb.{ext}` 的命名格式，其中：
- `{filename}`：目标文件或目标目录的完整文件名（不含扩展名）
- `{ext}`：图片文件的扩展名，仅允许以下小写值：`png`、`jpg`、`jpeg`、`gif`、`svg`

缩略图文件与目标文件位于同一层级，用于为目标对象提供可视化预览。在文件浏览器中，缩略图通常作为目标文件的预览图展示，提升用户体验。

### 元数据文件

元数据文件采用 `{filename}.meta.toml` 的命名格式，采用 [TOML](https://toml.io/) 格式存储结构化信息。

元数据文件与目标文件位于同一层级，为目标对象提供详细的描述信息。系统可读取这些元数据用于搜索索引、详情展示、分类筛选等功能。

## 文件层级关系

### 必需性规则

缩略图文件和元数据文件与其对应的目标文件遵循以下规则：

| 关系类型 | 说明 |
|---------|------|
| 位置要求 | 缩略图文件和元数据文件必须与目标文件在同一目录层级 |
| 存在性 | 缩略图文件和信息描述文件必定存在对应的目标文件或目标目录 |
| 可选性 | 对于目标文件而言，缩略图文件和信息描述文件是可选的，非必需 |

### 层级结构示例

```
[FILE_VIEW] 视频库/
├── 视频文件1.mp4
├── 视频文件1.mp4.thumb.jpg          # 缩略图
├── 视频文件1.mp4.meta.toml           # 元数据
├── 视频文件2.mp4
├── 视频文件2.mp4.thumb.png           # 缩略图
├── 视频文件2.mp4.meta.toml           # 元数据
├── 子目录/                           # 子目录
│   ├── 子目录文件.pdf
│   ├── 子目录文件.pdf.thumb.jpg      # 缩略图
│   └── 子目录文件.pdf.meta.toml      # 元数据
├── 子目录.meta.toml                  # 目录元数据（与子目录同级）
└── 子目录.thumb.png                  # 目录缩略图（与子目录同级）

[hide] 私有目录/                      # 隐藏目录
└── 内部文件.dat

[hide] 隐藏文件.txt                    # 隐藏文件
```

## 目录的元数据与缩略图

`file-view` 规范不仅适用于文件，也适用于目录。目录同样可以拥有元数据文件和缩略图文件，用于为目录提供标题、描述、标签等元信息，以及目录的预览缩略图。

### 目录的元数据文件

目录的元数据文件命名格式为 `{directory_name}.meta.toml`，与目录文件位于同一父级目录中，而非目录内部。目录元数据文件采用与文件元数据相同的 TOML 格式规范，包含 `info` 和 `extends` 两个字段。

### 目录的缩略图文件

目录的缩略图文件命名格式为 `{directory_name}.thumb.{ext}`，与目录文件位于同一父级目录中。目录缩略图通常用于展示目录的整体预览或代表图片。

## 元数据文件格式规范

元数据文件采用 TOML 格式，具有以下预定义字段结构：

### 字段定义

#### info 对象

`info` 是一个可选的对象类型字段，用于存储目标文件的基本描述信息：

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| `info.title` | string | 否 | 标题信息，用于覆盖或补充文件原名 |
| `info.describe` | string | 否 | 描述信息，提供目标内容的详细说明 |
| `info.tags` | string[] | 否 | 标签数组，用于分类和检索 |
| `links` | { title: string, url: string }[] | 否 | 相关链接数组，用于提供外部资源访问 |

#### extends 对象

`extends` 是一个可选的对象类型字段，用于存储扩展信息：
- 键（key）：字符串类型
- 值（value）：任意类型

该字段可用于存储与应用特定相关的数据，不受本规范限制，为扩展功能提供灵活性。

### 元数据文件示例

```toml
[info]
title = "示例视频标题"
describe = "这是一段关于xxx的详细描述信息"
tags = ["标签1", "标签2", "标签3"]

[extends]
custom_field = "自定义值"
version = 1
enabled = true

[[links]]
title = "官方网站"
url = "https://example.com"

[[links]]
title = "下载链接"
url = "https://download.example.com/file"
```

## 应用场景

本目录规范适用于以下典型场景：

- **多媒体库管理**：为图片、视频、音频文件提供缩略图和元数据支持
- **文档索引系统**：为文档文件添加标题、描述和标签，便于检索
- **内容展示平台**：为文件浏览器提供丰富的预览和描述信息
- **私有存储方案**：通过隐藏机制实现用户可见内容的灵活控制

## 实现建议

在实际实现中，建议遵循以下原则：

1. **自动生成**：程序应能自动为新增文件生成对应的缩略图和元数据文件
2. **一致性检查**：定期扫描目录，确保缩略图和元数据文件与目标文件的对应关系完整
3. **向后兼容**：新版本程序应能正确读取旧版本生成的元数据文件
4. **性能优化**：对于大量文件的场景，考虑使用异步方式处理缩略图生成和元数据写入
